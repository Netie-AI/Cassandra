# EVENT_MODULE_SPEC (Part 2 — deferred)

Build **after** Phase 9 is stable. See `BUILD.md` Part 2.

## Purpose

Event-driven anticipatory catalyst module: catalyst calendar + thesis tracker for second-order beneficiaries (e.g. OpenAI must pick AMD; Google antitrust Chrome window; Buffett discloses → infer who).

## Handoff to options engine

When a dated catalyst exists within expiry, set `has_dated_catalyst=True` on `ThesisView` so NEAR horizon can pass the gate.

## Honesty requirements (non-negotiable)

- "Bet every day until the event" is a theta trap — model cumulative bleed
- When catalyst window is too far for daily near-dated buying to be +EV, push to dated or longer-horizon structure
- Surfaces and gates anticipatory ideas; never auto-fires them
